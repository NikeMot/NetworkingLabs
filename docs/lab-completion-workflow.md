# Lab Completion Workflow

## Purpose

This document defines how labs should be completed in this repository.

The learner should focus on solving the lab. Documentation should be produced after the practical work is complete.

## Two-part lab requirement

Every lab has two required parts:

```text
Part 1 - New chapter or lab-specific content
Part 2 - Cumulative repetition of all topics learned so far
```

The learner must complete both parts before the final write-up is produced.

## Workflow

1. The lab is created and uploaded to the correct topic folder.
2. The learner completes Part 1: the new chapter or lab-specific content.
3. The learner completes Part 2: cumulative repetition of all topics learned so far.
4. The learner shares evidence for both parts: commands used, relevant outputs, files created or changed, issues encountered, and final verification results.
5. The documentation is written from the evidence.
6. Exactly seven reflection questions are asked.
7. The learner answers the seven reflection questions.
8. The final lab document is updated and uploaded to the correct topic folder.

## Reflection rule

Ask exactly seven reflection questions for each completed lab.

The standard questions are:

1. What problem did Part 1 of this lab simulate, and what new concept did it introduce?
2. What earlier topics did Part 2 force you to repeat, and which one felt weakest?
3. What evidence proved that both parts worked?
4. What issue, mistake, or confusing point did you encounter, and how did you resolve it?
5. What would you monitor, log, or alert on in production?
6. What would you improve if this were a real production environment?
7. What did you learn from this lab that you could explain to another engineer?

## Documentation rule

The final lab write-up should be concise, evidence-based, and production-focused.

The learner should not need to manually write the full documentation. Their responsibility is to complete both lab parts, provide evidence, and answer the reflection questions.

## Upload rule

Every completed lab must be uploaded to the correct numbered topic folder using the repository naming convention.
