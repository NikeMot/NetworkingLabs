# Target Role Readiness Map

## Purpose

This document maps the 24-lab curriculum to the target network engineering, network automation, observability, tooling strategy, and consulting roles.

The goal is not only to learn networking theory, but to produce evidence that demonstrates enterprise troubleshooting, architecture, design, automation, observability, documentation, and operational judgement.

## Target role outcomes

By the end of the 24 labs, the learner should have portfolio evidence for:

- advanced network troubleshooting and diagnostics
- enterprise networking architecture and design
- LAN, WAN, and Wi-Fi concepts and practical evidence
- network performance and availability management
- cost optimisation and solution design thinking
- CCNA-to-CCNP-level practical networking foundations
- Cisco, Juniper, and Palo Alto style firewall and network design concepts
- network automation and validation
- tooling landscape assessment
- target-state tooling strategy
- observability capability design
- CMDB and source-of-truth thinking
- cross-team documentation and consulting-style communication

## Required portfolio artefacts

The final portfolio should contain:

1. A completed lab write-up for each of the 24 labs.
2. Network diagrams for architecture, traffic flow, and failure scenarios.
3. Packet capture or command-output summaries where relevant.
4. Troubleshooting decision trees.
5. Monitoring and alerting recommendations.
6. Security and firewall policy reasoning.
7. Automation scripts or validation tooling.
8. Tooling landscape and target-state tooling strategy artefacts.
9. CMDB or source-of-truth model examples.
10. Seven-question reflections for each lab.

## Readiness caveat

This curriculum can produce strong portfolio evidence and interview talking points for network engineering, infrastructure, SRE, network automation, and consulting roles.

Real job readiness still depends on execution quality, depth of troubleshooting, quality of documentation, certification progress, and ability to explain design trade-offs clearly.
